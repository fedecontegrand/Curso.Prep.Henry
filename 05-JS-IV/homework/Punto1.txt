Objeto: es un elemento en el cual en vez de guardar informacion diversa como una matriz, guarda mucha informacion referida a una sola cosa. Esta compuesto por las keys y sus respectivos valores. Las keys no pueden tener el mismo nombre entre ellas, pero los valores de diferentes keys si pueden coincidir entre si. Una key vendria a ser el equivalente del index de una matriz, pero en este caso nosotros no la reconocemos por su ubicacion, sino que por su nombre. Los objetos se escriben entre corchetes "const objeto={}". 
Propiedades: son las keys(claves) que le asignamos al objeto. Las podemos citar con notacion de puntos o de corchetes, veremos la diferencia mas abajo.
Metodos: cuando al valor de las keys del objeto la definimos a traves de una funcion, lo denominamos "Metodo"
bucle for..in: a diferencia de las matrices, donde podemos iterar mas facilmente ya que cada elemento le corresponde un index segun su posicion, en un objeto para iterar deberemos utilizar el bucle "for ... in". Se escribe "for(let...in...){..." donde despues del let introducimos clave para que recorra todas las claves, y luego del in colocamos el nombre del objeto, para que sepa a que elemento va a iterar.
Notacion de puntos vs notacion de corchetes: la notacion de puntos es mas simple y sirve para citar la clave del objeto a la que hacemos referencia de manera simple y directa. Si vamos a usar corchetes deberemos mencionar el nombre de la clave a la que hacemos referencia entre comillas, como a una string. Los corchetes se usan mas que nada cuando tenemos una variable que pueda apuntar a una de las keys del objeto, en ese caso, no se utiliza las comillas.
